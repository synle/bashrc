# bashrc ‚Äî Architecture

## High-Level Overview

`synle/bashrc` is a personal dotfiles and shell-environment bootstrapper that targets macOS, Ubuntu/Debian/Mint, Arch/SteamOS, RHEL/Fedora, Windows (PowerShell, MinGW, WSL), ChromeOS, and Android Termux from a single codebase. The setup engine is a Node.js ‚Üí bash pipeline: a top-level orchestrator (`run.sh`, driven by `Makefile`) sources environment detection, then runs `cat software/index.js | node | bash`. `index.js` discovers per-tool scripts in `software/scripts/`, emits bash commands to stdout, and bash executes them in sequence ‚Äî installing tools, writing config files, and assembling `~/.bash_syle` from `# BEGIN`/`# END` marker blocks.

The repo also ships a Vite + React single-page webapp (in `webapp/`, sources also in `src/`) that renders the demo / installer page at `https://synle.github.io/bashrc/`, plus a Vitest test suite under `software/tests/` and `tests/`. Build artifacts (assembled profile scripts, editor configs, host files) land in `.build/` and are regenerated by CI across five OS runners in parallel.

## Important Files

- **`run.sh`** ‚Äî Main user-facing entrypoint. Detects local vs remote mode, parses flags (`--files`, `--preset`, `--debug`, `--refresh`, `--dryrun`, `--remove`), sources `common-env.sh`, then drives the Node-to-bash pipeline.
- **`Makefile`** ‚Äî GNU make targets for setup (`setup`, `setup_local_full`, `setup_prod`), validation (`validate`, `test_all`, `dry_run`, `doctor`), and CI phases (`ci_prep`, `ci_test_unit`, `ci_test_profile`, `ci_test_smoke`, `ci_test_buildconfig`, `ci_test_coverage`).
- **`software/index.js`** ‚Äî Dual-purpose: shared utility library (parsers, file I/O, network, platform detection, console output, tagged template literals like `code`, `list`, `json`, `readText`, `readJson`) AND the bootstrap orchestrator that discovers and runs scripts under `software/scripts/`.
- **`software/common.js`** ‚Äî Common helpers shared with individual scripts.
- **`software/scripts/advanced/llm/`** ‚Äî Per-CLI AI client setup (`claude/`, `copilot/`, `gemini/`, `opencode/`, plus `ollama.sh` / `ollama.profile.bash`). Shared rules live in `_common/` (the canonical `instructions.md` is the single source for the global `~/.claude/CLAUDE.md`, `~/.copilot/AGENTS.md`, and `~/.gemini/GEMINI.md` files; `_common/commands/*.md` feeds Claude's slash-command set, which OpenCode mirrors via symlinks); `llm-common.js` exposes shared JS helpers including `getOllamaProviderInputs()` (probes both the workstation and `127.0.0.1` for Ollama models). The full surface-parity matrix + settings-intent table lives in [`software/scripts/advanced/llm/_common/README.md`](software/scripts/advanced/llm/_common/README.md). Run all four CLIs in one shot with `bash run.sh --preset=llm`.
- **`software/bootstrap/common-env.sh`** ‚Äî Sets `REPO_PATH_IDENTIFIER`, `REPO_BRANCH_NAME`, `ALL_OS_FLAGS`, and other env consumed by both the engine and `vite.config.js`.
- **`software/scripts/_init.js` / `~cleanup.js` / `~wrapup.sh`** ‚Äî Lifecycle hooks bracketing every run (init writes preamble; cleanup strips unfilled markers and finalizes `~/.bash_syle`).
- **`software/tools/build-include.js`** ‚Äî Implements the marker-based include system: inlines file content between `# BEGIN path` / `# END path` markers and resolves `// {{theme.key}}` color tokens in JSONC.
- **`install.sh`** ‚Äî Public installer wrapper users `curl | bash`.
- **`package.json`** ‚Äî Declares webapp/test scripts (`dev`, `build`, `test`, `test:smoke`, `test:profile`, `test:buildconfig`, `format`) and pins React 19, Vite 4, Vitest 4, Prettier, Puppeteer, Sass.
- **`vite.config.js`** ‚Äî Vite config rooted at `webapp/`, single-file output to `dist/`, with a custom plugin that timestamps the service worker.
- **`vitest.config.js` / `vitest.smoke.config.js` / `vitest.profile.config.js` / `vitest.buildconfig.config.js` / `vitest.smoke.local.config.js`** ‚Äî One Vitest config per suite.
- **`webapp/index.jsx`, `webapp/index.scss`, `webapp/sw-bashrc.js`** ‚Äî React entry, styles, and service worker for the demo site.
- **`CLAUDE.md`** ‚Äî Repo-specific instructions for Claude / coding agents (large; project conventions).
- **`DEV.md`** ‚Äî Developer quick-start (install deps, run dev server, run setup, run tests, build).
- **`README.md`** ‚Äî User-facing overview, highlights, install one-liner, and `run.sh` flag reference.

## Build & Release Flow

CI runs in `.github/workflows/build-main.yml`, phased:

1. **Prep** (Ubuntu) ‚Äî `make ci_prep` runs Prettier + `build-include` + the webapp build. Uploads a `prep-patch` artifact and `webapp-dist`.
2. **Build (parallel √ó 5)** ‚Äî `build-ubuntu`, `build-rhel` (Fedora container), `build-arch` (Arch container), `build-debian` (Debian container), `build-mac` (macOS). Each applies the prep patch, runs `ci-build`, uploads its `.build/` as `build-<os>`, uploads the per-OS `profile_bashrc_<os>.sh` as `profile-build-<os>` (separate artifact), and on default-branch pushes mirrors the profile to the `binary-cache` release on `synle/bashrc` as `bashrc-profile__profile_bashrc_<os>.sh`. Build jobs declare `permissions: contents: write` to enable the release upload.
3. **Publish** ‚Äî Merges per-OS `.build/` artifacts into `.build/` (excluding `profile_bashrc_*.sh`, which is not tracked in git), amends HEAD with `--force-with-lease` (note: requires `fetch-depth: 0`), downloads the `profile-build-*` artifacts and copies each `profile_bashrc_<os>.sh` into `dist/` so the deployed Pages site serves them same-origin for the webapp, deploys `dist/` to GitHub Pages, and mirrors the run as a commit status on the amended SHA.
4. **Test** ‚Äî Runs `ci_test_unit`, `ci_test_profile`, `ci_test_smoke`, `ci_test_buildconfig`, `ci_test_coverage`; renders coverage and a per-package install matrix into the job summary; tags successful pushes as `v0.YYYYMMDD.<run_number>`.

Supporting workflows: `cleanup-artifacts.yml` and `cleanup-pr-artifacts.yml` prune old artifacts. There is no app-style release workflow ‚Äî distribution is via the rolling `main` branch consumed by `curl | bash` and via the GitHub Pages deployment of `webapp/`.
